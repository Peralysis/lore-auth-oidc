# Authentik

Create an OAuth2/OpenID Provider and associate it with an Authentik application.

1. Select a confidential client and authorization-code flow.
2. Add the exact `OIDC_REDIRECT_URL` as a strict redirect URI.
3. Assign the `openid`, `profile`, and `email` scope mappings.
4. Configure the provider slug in the issuer URL:

```dotenv
OIDC_ISSUER_URL=https://identity.example.com/application/o/lore/
OIDC_SCOPES=openid profile email
OIDC_PROVIDER_NAME=authentik
```

The issuer includes the application/provider slug and trailing slash. Copy the client ID and secret generated by Authentik. Default profile mappings normally provide the standard name and username claims.

Reference: [Authentik OAuth2/OpenID Provider](https://docs.goauthentik.io/add-secure-apps/providers/oauth2/).
